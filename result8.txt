
 Model: 1 
Model: I got you!

Background: The project aims at protecting owner's property by handling intruders. The three epics for this project are Detect Intruder, Notify Stakeholder, and Response to intrusion. The project should be accurate, safe, and timely.

Epic 1: Detect Intruder
Security Manager:
- As a security manager, I can receive alerts when an intruder is detected, so that I can take immediate action.
- As a security manager, I can see a live video feed of the intruder, so that I can quickly assess the situation and decide on the correct response.

Epic 2: Notify Stakeholder
Visitor:
- As a visitor, I can be alerted when there is an intrusion, so that I can take evasive action or seek help.
- As a visitor, I can alert security when I see an intruder, so that they can respond appropriately.

Police:
- As the police, I can be alerted when there is an intrusion, so that I can dispatch officers to the scene.
- As the police, I can see a live video feed of the intrusion, so that I can assess the situation and bring the correct resources.

Place Owner:
- As the place owner, I can receive alerts when there is an intrusion, so that I can take action.
- As the place owner, I can see a log of intrusions and responses, so that I can learn from the data and improve future responses.

Epic 3: Response to Intrusion
Security Manager:
- As a security manager, I can lock down the building when I detect an intruder, so that they cannot move around freely.
- As a security manager, I can evacuate visitors when there is an intrusion, so that they are not harmed.

Police:
- As the police, I can arrive at the scene of the intrusion within a certain timeframe, so that I can control the situation and apprehend the intruder.
- As the police, I can collaborate with security to plan the best response to the intrusion, so that we can work together to neutralize the threat.

Intruder:
- As an intruder, I can be deterred from breaking in, so that I do not engage in criminal activity.
- As an intruder, I can be identified and caught quickly and efficiently, so that I cannot cause any harm.

Model: Restaurant Review Web App

Epic 1: View Reviews
Guest:
- As a guest, I can view restaurant reviews, so that I can decide whether to visit that restaurant or not and feel reassured by seeing other people's experiences.

User:
- As a user, I can view my own past reviews, so that I can easily remember which restaurants I have already reviewed.

Epic 2: Post Reviews
Guest:
- As a guest, I can create a new review about a restaurant, so that I can share my experience with others and feel like I am contributing to the community.

User:
- As a user, I can edit or delete my own reviews, so that I can correct any mistakes or update my review if something changes.

Epic 3: Edit Reviews
Guest:
- As a guest, I cannot edit or delete any reviews because I do not have an account.

User:
- As a user, I can report a review that violates the community guidelines, so that I can help maintain a fair and respectful environment for everyone.

Epic 4: Search Reviews
Guest:
- As a guest, I can search for reviews based on keywords, location, or cuisine, so that I can find the information I need quickly and efficiently.

User:
- As a user, I can filter the reviews to only see the ones I am interested in, so that I can easily find the most relevant reviews for me.

Stakeholders:
- Guests: People who visit the website without creating an account.
- Users: People who have created an account and are logged in.

Model: Retail/Hire Web application
Epic 1: Package Management
Staff:
- As a staff member, I can easily add new products to the system, so that I can keep the inventory up-to-date and provide customers with the latest offerings.
- As a staff member, I can manage product descriptions and photos, so that I can make sure all product information is clear and accurate, and visually appealing for customers.
- As a staff member, I can track stock levels, so that I can ensure that products are always available for customers to purchase.

Customer:
- As a customer, I can easily browse available products, so that I can find what I need quickly and easily.
- As a customer, I can filter products based on various criteria (e.g. price, category, popularity), so that I can easily narrow down my options to find exactly what I want.
- As a customer, I can add products to my shopping cart and checkout, so that I can purchase products easily and securely.

Epic 2: Calendar Functionality
Staff:
- As a staff member, I can view an overview of upcoming orders, so that I can plan inventory and prepare orders on time.
- As a staff member, I can set and manage delivery dates, so that I can ensure orders are delivered on time and efficiently.
- As a staff member, I can track the status of orders, so that I can keep customers informed and resolve any issues quickly.

Customer:
- As a customer, I can select a convenient delivery date and time, so that I can receive my order at a time that works for me.
- As a customer, I can track the status of my order, so that I can stay informed about when it will be delivered.
- As a customer, I can easily reschedule or cancel a delivery, so that I can adjust my plans as needed.

Emotional Goals integrated in user stories:
- Readable: Customers can easily browse and filter products based on their preferences.
- Understandable: Product descriptions and photos are clear and accurate, making it easy for customers to make informed purchasing decisions.
- Self-learnable: Customers can easily navigate the site and make purchases without needing extensive help or guidance.
- Precise on number: Staff can track inventory levels accurately to ensure products are always available for customers to purchase.
- Up-to-date: Staff can easily add new products to the system and update existing products to keep the inventory up-to-date.
- Easy-to-use: Both staff and customers can navigate the system easily and efficiently to manage orders and make purchases.
